# Next Steps for Policy-as-Code Agent

This document outlines the planned future enhancements for the Policy-as-Code Agent.

## 1. Create Evaluation Datasets (Completed)

- **Objective:** Expand the test suite to cover a wider range of policy types and edge cases.
- **Tasks:**
    - **Done:** Created a new "Data Freshness" test category.
    - **Done:** Added `test_data_freshness.py` and `test_data_freshness.yaml`.
    - **Done:** Implemented `check_policy_local` in `agent.py` to facilitate local testing.

## 2. Integrate with Dataplex APIs for Policy Enforcement (Completed)

- **Objective:** Move beyond policy validation to automated policy enforcement.
- **Tasks:**
    - **Structured Remediation:** Update the remediation prompt to generate structured, machine-readable remediation steps (e.g., JSON).
    - **Enforcement Tool:** Create a new `enforce_remediation` tool that can parse these structured steps and call the appropriate Dataplex API to apply the fixes (e.g., adding a missing label to a table).
    - **Permissions and Authentication:** Ensure the agent has the necessary IAM permissions to modify Dataplex resources and that the API calls are properly authenticated.

## 3. Advanced Memory Features (Completed)

- **Objective:** Provide the agent with a mechanism to store and retrieve information, both for caching and for maintaining conversational context.
- **Tasks:**
    - **Done:** Implemented a persistent, scalable memory using **Google Cloud Firestore**.
    - **Done:** Integrated **Vector Search** for semantic policy retrieval.
    - **Done:** Implemented policy versioning.
    - **Done:** Implemented memory pruning.
    - **Done:** Implemented enhanced search and discovery (author, date range).
    - **Done:** Implemented user feedback and policy ranking.
    - **Done:** Implemented analytics and history tracking (Execution History, Top Violations, Resource Search).

## 5. Security & Sandboxing (Completed)

- **Objective:** Enhance the security of the agent by preventing the execution of malicious or harmful code generated by the LLM.
- **Tasks:**
    - **Done:** **Restricted Execution:** Modify `run_simulation` to pass a restricted dictionary of globals (e.g., only allowing `json`, `re`, `datetime`) instead of the full `globals()`.
    - **Done:** **AST Analysis:** Before executing, parse the generated code with Python's `ast` module to detect and reject usage of dangerous modules like `os`, `sys`, `subprocess`, or file I/O operations.
    - **Code Review Tool:** Add a "Dry Run" tool or step that explains the generated code in plain English to the user before execution.
    - **Sandboxed Environment:** Investigate and implement a more robust sandboxing solution (e.g., Docker-based execution or Google's Code Interpreter) for complete isolation.

## 6. Reliability & Testing

- **Objective:** Ensure the agent generates correct and robust policy code before running it on real data.
- **Tasks:**
    - **Self-Correction Loop:** Instruct the LLM to generate a *test case* (small fake metadata snippet) along with the policy code. The agent should run this test case first. If it fails, the agent should self-correct before processing the user's actual data.
    - **Configuration Validation:** Add startup checks to verify that critical environment variables (like `FIRESTORE_DATABASE`, `GCP_REGION`) are set, preventing runtime errors.

## 7. New Features

- **Objective:** Expand the agent's capabilities to provide more value to the user.
- **Tasks:**
    - **Active Enforcement (Tagging):** Add a tool to automatically apply a tag (e.g., `policy_violation: true`) to assets in Dataplex that fail validation.
    - **Done:** **Compliance Scorecard:** Create a `generate_compliance_scorecard` tool that runs a suite of "Core Policies" against a dataset and calculates a compliance percentage.
    - **Done:** **Rich Reporting:** Add a tool to export violation reports as HTML or CSV files for easier consumption of large result sets.

## 8. Architectural Improvements (Partially Completed)

- **Objective:** Improve the code quality, maintainability, and structure of the agent.
- **Tasks:**
    - **Done:** **Refactoring:** Extract common logic from `run_policy_from_gcs` and `run_policy_on_dataplex` (specifically error handling, logging, and result formatting) into a shared helper function to reduce code duplication.
    - **Conversational Refactor:** Rebuild the agent's interaction model to be fully conversational using `adk.Runner` and session-based context.

## 9. Authentication & Identity

- **Objective:** Personalize agent interactions and secure access by integrating a robust identity provider.
- **Tasks:**
    - **Frontend Integration:** Implement "Sign-In with Google" on the client-side application using Google Identity Services to authenticate users and retrieve ID tokens.
    - **Backend Verification:** Implement server-side logic to verify the Google ID token's signature and expiration before processing any requests.
    - **Session Management:** Initialize agent sessions using the verified Google `sub` (unique user ID) instead of generic or hardcoded identifiers to ensure memory isolation and personalization.
